## 1. Формы и обратная связь

### 1.1. Форма «Записаться» (главная)

**Где:** на главной, в шапке (кнопка-CTA), в футере, на каждой странице направления, на странице специалиста, после статьи в блоге.

**Поля:**
- Имя (обязательно)
- Телефон (обязательно, валидация)
- Telegram / WhatsApp (опционально)
- Тема запроса (опционально, textarea)
- Скрытое поле: «Откуда пришёл» (страница, где была заполнена)
- Скрытое поле: «На какого специалиста» (если форма со страницы специалиста — подставляется имя)
- Чек-бокс согласия на обработку ПД (обязательно)

**После отправки:**
1. Сообщение в Telegram-бот администратора (мгновенно)
2. Email-уведомление администратору (резерв)
3. Запись в Яндекс.Метрику как достигнутая цель
4. Пользователь видит экран благодарности с инструкцией: «Центр свяжется с вами в течение 30 минут в рабочее время»
5. Опционально (поздний этап): автосообщение клиенту в Telegram/WhatsApp с подтверждением

### 1.2. Форма «Записаться на диагностику 500₽»

Упрощённая версия — только имя и телефон. Самый низкорисковый шаг, без блокирующих полей.

### 1.3. Виджет «Написать нам» (плавающая кнопка)

**Где:** на всех страницах сайта, фиксированная иконка в правом нижнем углу (или боковая панель).

**Формат:** небольшой попап при клике.

**Поля:**
- Имя (обязательно)
- Телефон (обязательно)
- Чек-бокс согласия на обработку ПД

**После отправки:** уведомление администратору в Telegram + запись в Метрику. Пользователь видит: «Мы перезвоним вам в течение 30 минут».

**Назначение:** альтернативный путь для тех, кто не готов сразу идти в YClients — просто оставить контакт.

### 1.4. Форма обратной связи (на /contacts)

Стандартная: имя, контакт, сообщение.

### 1.4. Подписка на полезное

Простая форма «Получать письма с полезным от психологов центра» — имя + email или Telegram. Поздний этап.

## 2. Каталог специалистов с фильтрами

**Центральный функционал сайта — логика интернет-магазина.**

**Главный референс:** [palehova.ru](https://palehova.ru/) — «Каталог психологов по цене и отзывам». Воспроизводим логику и принципы UX, не дизайн.

Три ключевых механизма работают одновременно (пересекаются):
1. **Поиск по запросу** — текстовый поиск по симптомам, методам, темам.
2. **Фильтрация** — сужение списка по характеристикам.
3. **Сортировка** — по цене, по отзывам, по опыту.

### Поиск по запросу (новое — добавлено 2026-05-15)

Поле сверху каталога: **«Что вас беспокоит?»**

- Поиск по тегам симптомов («тревога», «выгорание», «панические атаки», «подросток молчит»), методам («КПТ», «расстановки», «гештальт»), темам («отношения», «деньги», «самооценка»).
- Подсказки появляются по мере ввода (autocomplete).
- При выборе подсказки — соответствующий фильтр в каталоге активируется автоматически.
- Поиск понимает синонимы (например, «панические атаки» = «панические приступы» = «ПА»).

### Сортировка (новое — добавлено 2026-05-15)

Выпадающее меню справа от фильтров:

- **«Рекомендуем»** (по умолчанию) — порядок задаёт администратор центра вручную через админку. Позволяет продвигать приоритетных специалистов.
- **По цене ↑** (от дешёвых к дорогим)
- **По цене ↓** (от дорогих к дешёвым)
- **По рейтингу** — средняя оценка отзывов на Яндекс.Бизнес
- **По количеству отзывов** — больше отзывов = выше
- **По опыту** — больше лет работы = выше

### Логика фильтрации

Все фильтры работают **И** (пересечение). Если выбрано «семейный психолог» + «онлайн» + «женщина» — выводятся специалисты, которые **И** семейные **И** работают онлайн **И** женщины.

### Параметры фильтрации

| Фильтр | Тип | Значения |
|---|---|---|
| **Направление работы** | мультивыбор | детский, подростковый, семейный, клинический, психотерапевт, перинатальный, расстановки, групповая работа, бизнес-расстановки |
| **Возраст клиентов** | мультивыбор | дети 4–10, подростки 11–17, взрослые, пары, семьи |
| **Формат** | мультивыбор | очно, онлайн |
| **Пол специалиста** | одиночный | женщина, мужчина, неважно |
| **Опыт работы** | одиночный | от 5 лет, от 10 лет, от 15 лет, неважно |
| **Цена** | диапазон | 2 500 — 7 500 ₽ |
| **Филиал** | одиночный | Домодедово (пока единственный, ждёт расширения) |

### Сортировка

- По умолчанию: специально подобранный порядок (администратор отмечает «приоритетных» специалистов в админке)
- По имени (А–Я)
- По цене (возрастание / убывание)
- По опыту (убывание)

### URL и SEO

Каждая комбинация фильтров формирует URL вида `/specialists?direction=family&format=online`. Для SEO — статические страницы под популярные комбинации:
- `/specialists/family` → «Семейные психологи Москва / Домодедово»
- `/specialists/kids` → «Детские психологи»
- `/specialists/online` → «Онлайн психолог»

Title и meta description на этих страницах генерируются по шаблону.

## 3. Запись на консультацию — три канала (принято 2026-05-14)

Сайт предоставляет клиенту три способа записи — на выбор.

### Канал 1 — YClients-виджет (основной)

Виджет YClients размещён на странице специалиста и подключён к главным CTA «Записаться»:
- Клиент выбирает специалиста, дату, время → запись попадает сразу в YClients
- Администратор подтверждает в течение часа
- Клиент получает уведомление

Технически — JS-виджет YClients (бесплатный, готовый код-сниппет от вендора). Подключается на этапе 1.

### Канал 2 — Кнопка «Позвонить»

Заметная кнопка с телефоном `+7 929 589-99-05` в шапке, в hero-блоке и на странице специалиста. На мобильном — `tel:` ссылка, клик сразу инициирует звонок.

### Канал 3 — Форма обратной связи

Простая форма (имя, телефон, тема). После отправки:
1. Письмо администратору на email (главный канал)
2. Дублирующее уведомление в WhatsApp администратору
3. Запись в Яндекс.Метрику как достигнутая цель

### Telegram не используется как канал записи

Telegram в РФ заблокирован, работает только через VPN или прокси. Соответственно:
- Основной CTA «Написать в Telegram» не размещается
- Виджет Telegram-чата на сайте не используется
- Telegram-канал @vremya_psy упоминается в футере как соцсеть для контента, не как канал связи
- Для тех, кому удобнее Telegram, ссылка размещается в блоке соцсетей, не как первичный способ записи

WhatsApp и VK работают в РФ штатно и используются как мессенджеры для общения с клиентом.

## 4. Мультифилиальная архитектура

**Архитектурное решение, закладываемое со старта**, несмотря на единственный филиал на текущий момент. Это исключает переписывание сайта при появлении новых филиалов.

**Главный референс:** [waxandnails.ru](https://waxandnails.ru/) → [zelenograd.waxandnails.ru](https://zelenograd.waxandnails.ru/) — сеть с отдельными поддоменами на каждый город. Воспроизводим принцип, не дизайн.

**Как устроено у Wax & Nails:**
- Главный `waxandnails.ru` — лендинг бренда, кнопка ведёт на страницу `/salon` со списком филиалов
- Каждый филиал — на собственном поддомене (`zelenograd.waxandnails.ru`, и т.д.)
- Внутри филиала — собственные услуги, цены, расписание, контакты, специалисты

### Три технических варианта (выбирает разработчик)

| Вариант | Плюсы | Минусы |
|---|---|---|
| **Поддомены** (`domodedovo.psyau.ru`) | Как у Wax & Nails. Лучшее SEO под локальные запросы. Каждый филиал — самостоятельный сайт. | Сложнее технически: настройка DNS, wildcard SSL, отдельная конфигурация на филиал. |
| **Префиксы URL** (`psyau.ru/domodedovo/...`) | Один домен — один SSL. SEO работает почти как с поддоменами. Проще админка. | Менее «масштабно» визуально, чем поддомены. |
| **Один домен + cookie** (selector в шапке переключает контент) | Самый простой. Один сайт. | Хуже SEO под локальные запросы. Поисковики не различают филиалы. |

Решение принимает разработчик. В наших документах используем абстрактную формулировку «выбранный филиал» — она работает для любого из трёх вариантов.

### Что это значит технически

**В базе данных у каждой сущности есть поле `branch_id`:**
- Специалист принадлежит одному или нескольким филиалам
- Событие в афише привязано к филиалу
- Расписание зависит от филиала
- Цены могут отличаться по филиалам

**На сайте:**

| Где | Как работает |
|---|---|
| Шапка сайта | Selector филиала (выпадающий список) → меняется контент. Сохраняется в cookie на 30 дней. |
| Главная страница | Контент адаптируется под выбранный филиал (специалисты, события). Контакты и адрес — соответствующие. |
| Каталог специалистов | Показываются специалисты выбранного филиала + специалисты «онлайн» (они вне филиала) |
| Афиша | События выбранного филиала + онлайн-события |
| Контакты | Адрес выбранного филиала + общие телефоны |

### Если выбран «онлайн / неважно»

Отображаются все специалисты, работающие онлайн. Адрес скрывается.

### Стартовое состояние

Один филиал — Домодедово. Selector активен, но в списке один пункт. При добавлении Москвы / СПб новая запись заводится в админке, правки кода не требуются.

## 5. Блог-движок

### Требования

- Создание / редактирование статей в админке
- Поля статьи: заголовок, slug, автор (привязка к специалисту), тема (тег), обложка, краткое описание, тело (HTML/Markdown), дата публикации, статус (черновик / опубликовано), SEO-поля (title, description)
- Связи: статья → автор (специалист), статья → темы (мультивыбор), статья → связанные направления
- На карточке специалиста его статьи отображаются автоматически
- На странице направления отображаются связанные статьи

### Редактор для авторов

Простой markdown-редактор или WYSIWYG (CKEditor / TinyMCE). Если статью готовит редактор заказчика — готовый текст передаётся администратору для публикации.

### SEO

- Хлебные крошки: Главная → Блог → Тема → Статья
- Schema.org разметка: Article + Person (автор)
- Sitemap.xml автогенерация

## 5а. Конструктор психологических тестов

**Добавлено 2026-05-14.**

### Требования

- Создание / редактирование тестов в админке
- Поля теста: название, slug, обложка, краткое описание, время прохождения, автор-психолог (привязка), направление (привязка), статус (черновик / опубликовано), SEO-поля
- **Типы вопросов:**
  - Одиночный выбор (radio)
  - Мультивыбор (checkbox)
  - Шкала Лайкерта (1–5, «совсем нет» — «очень сильно»)
  - Да/Нет
- **Логика подсчёта:** каждому ответу — балл. Сумма баллов → диапазон → интерпретация
- **Интерпретации:** текстовый результат для каждого диапазона + рекомендация специалиста + ссылка на услугу
- **Сохранение прогресса** в localStorage (если ушёл — вернётся)
- **Опционально на старте, обязательно на этапе 2:**
  - Захват email для отправки результата на почту
  - Анонимная аналитика (сколько прошли, какие результаты популярны)

### Технически

- На старте — без сложной математики, простой подсчёт суммы баллов
- Адаптивная вёрстка обязательна — большинство прохождений ожидается с мобильных устройств
- Результат не блокируется регистрацией (это снижает конверсию). Email — опционально.
- Защита от ботов (rate limit на отправку формы результата)

### SEO

Каждый тест — отдельная страница с полноценным SEO. Тесты — один из основных источников органического трафика по запросам «тест на …».

### Связь с воронкой

После прохождения — три варианта пути для клиента:
1. **Записаться на диагностику 500₽** (главный CTA)
2. **Записаться к рекомендованному специалисту** (по теме теста)
3. **Прочитать статью по теме** (мягкий путь через блог)

## 7. Отзывы

### Источники

- Яндекс.Бизнес (5,0 · 50 отзывов) — подгрузка через виджет Яндекса
- Внутренние отзывы из [отзывы/отзывы.md](../отзывы/отзывы.md) — добавляются в админке
- Возможность клиенту оставить отзыв через форму на сайте → модерация → публикация

### Где показываются

- На главной — карусель
- На странице специалиста — отзывы об этом специалисте
- На странице направления — отзывы по этому направлению
- На отдельной странице /reviews — все отзывы с фильтрами

### Структура отзыва

- Текст
- Имя / инициалы / аноним
- К какому специалисту (привязка)
- По какому направлению (тег)
- Дата
- Источник (Яндекс / сайт / Авито)
- Опционально: фото клиента (с согласия)

## 12. SEO-требования

Подробно — в [seo/стратегия.md](../seo/стратегия.md). Здесь технические:

- Уникальный title и meta description на каждой странице
- H1 один на страницу, осмысленный
- Семантические URL (без `?id=123`)
- ЧПУ для всех страниц (`/specialists/natalia-lukyanova`, не `/specialists?id=4`)
- Каноникал-теги
- robots.txt + sitemap.xml
- Open Graph + Twitter Cards для всех страниц
- Скорость загрузки ≤ 2 сек на мобильном (Core Web Vitals)
- Schema.org: LocalBusiness, Person (специалисты), Article (статьи), Event (события), Review (отзывы), FAQPage (FAQ)
